# Code Review

Perform a deep, high-value review of the current changes.

> **Related Rule**: `code-review/RULE.md`

## Steps

1. Identify all changed files
2. Review each change against the code-review rule checklist
3. Classify issues as BLOCKING or ENHANCEMENT
4. Provide actionable feedback with fixes

Focus on correctness, security, architecture, performance, and maintainability. Do NOT bikeshed on formatting or minor style preferences.
