# Generate Commit Message

Analyze the staged changes and generate a conventional commit message.

> **Related Rule**: `gitflow/RULE.md`

## Steps

1. Run `git diff --staged` to see changes
2. Identify the primary purpose of the changes
3. Determine the appropriate type and scope
4. Generate the commit message following the gitflow commit conventions

## Output

Generate the commit message based on the actual staged changes.
